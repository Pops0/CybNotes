# Correction + Answers

## 1️⃣ MFA Bypass

**Your answer:**

- Privilege escalation
    
- Not sure for the other
    

**Score:** 0.5 / 1

Privilege escalation usually happens **after** access is gained, so it’s not really an MFA bypass itself.

Two **very common real-world MFA bypasses** are:

**1. MFA Fatigue / Push Bombing**  
Attacker repeatedly sends MFA prompts until the user clicks **Approve** out of annoyance.

**2. Adversary-in-the-Middle Phishing (AiTM)**  
Tools like **Evilginx** or **Modlishka** proxy the login page.

What happens:

User → fake login page  
Attacker proxy → real site  
User completes MFA  
Attacker steals session cookie

The attacker logs in **without needing MFA again**.

This technique has been used in major breaches involving **Microsoft** and **Twilio** contractors.

---

## 2️⃣ Word Spawning PowerShell

**Your answer:**

> A script shouldn't run without user intervention.

**Score:** 0.5 / 1

You’re **thinking in the right direction** — this is abnormal behavior.

The real issue:

**Microsoft Word should NOT normally launch PowerShell.**

This usually indicates:

**Malicious Office macros**

Typical attack chain:
```
Phishing email  
↓  
Word document  
↓  
User enables macros  
↓  
Macro launches PowerShell  
↓  
PowerShell downloads malware
```

#MITRE
This technique maps to **MITRE ATT&CK technique T1059 (Command Execution)** and **T1204 (User Execution)**.

Security tools often flag it as:

> **"Office → PowerShell child process"**

Very common in ransomware attacks.

## 3️⃣ Leaked AWS Access Key

**Your answer:**

- Remove the key
    
- Refresh/renew the key
    

**Score:** 1 / 1 ✅

Excellent. This is exactly right.

Immediate response should be:

1️⃣ **Deactivate / delete the key immediately**  
2️⃣ **Rotate credentials (generate new keys)**

Then follow up with:

3️⃣ Review **AWS CloudTrail logs  
4️⃣ Check for malicious activity (crypto miners, new users, etc.)

Attackers **scan GitHub constantly** for exposed keys.

Some keys are abused **within minutes**.

## 5️⃣ CEO Gift Card Scam

**Your answer:**

**Score:** 1 / 1 ✅

Correct.

More formally this is:

**Business Email Compromise (BEC)**

or specifically

**CEO Fraud / Executive Impersonation**

The attacker relies on:

- Authority pressure
    
- Urgency

It’s extremely profitable and costs companies **billions every year** according to the **Federal Bureau of Investigation** Internet Crime Complaint Center.
