# Answers

## 1️⃣ Suspicious Login From New Country

**Your answer:**

> I have no idea

**Score:** 0 / 1

The control being described is **risk-based or conditional authentication**.

A common example is **Conditional Access** in  
Microsoft Entra ID.

These systems analyze signals like:

- location
    
- device
    
- login time
    
- IP reputation
    
- impossible travel
    

Example detection:

User logs in from South Africa  
↓  
5 minutes later login from Russia  
↓  
Impossible travel detected  
↓  
Access blocked or MFA required

This is called **risk-based authentication** or **adaptive authentication**.

## 3️⃣ PowerShell `-enc` Flag

**Your answer:**

> is it to do with encryption?

**Score:** 0.75 / 1

Very close.

The flag means **encoded command**.

Example:

powershell -enc SQBFAFgA

Attackers encode commands in **Base64** to:

- hide malicious commands
    
- bypass security filters
    
- evade detection tools
    

Example chain:

Phishing doc  
↓  
Macro launches PowerShell  
↓  
Encoded command executes malware

This is commonly detected in SOC alerts.

## 4️⃣ Blocking Inbound Traffic

**Your answer:**

> Most companies don’t need inbound traffic and only allow specific ports

**Score:** 1 / 1 ✅

Excellent answer.

You correctly described **default-deny inbound firewall policy**.

Typical reasoning:

Internet → Company Network  
HIGH RISK

So organizations only allow inbound traffic to **specific services** like:

- HTTPS (443)
    
- VPN portals
    
- mail gateways
    

Everything else stays blocked.
